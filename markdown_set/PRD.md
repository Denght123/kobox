# 追番收藏柜网站PRD(小匣子kobox PRD)

## 1. 文档信息
- 产品名称：小匣子kobox
- 产品形态：响应式 Web 网站
- 优先终端：PC 优先，兼顾移动端浏览
- 产品阶段：MVP 定义
- 核心方向：动漫收藏管理 + 个人主页展示 + 分享传播

---

## 2. 产品概述
### 2.1 一句话定位
一个帮助二次元用户收藏动漫、自动匹配官方封面，并把自己的动漫履历以“动漫收藏柜”形式展示和分享的网站。

### 2.2 产品本质
本产品不是社区，也不是追番进度工具，而是一个：
1. **动漫收藏管理工具**
2. **个人二次元主页展示产品**
3. **可传播的动漫收藏名片**

### 2.3 解决的问题
当前用户在整理自己动漫收藏时，通常会遇到：
- 看过很多番，但没有一个专门展示“我收藏了什么动漫”的地方。
- 想做自己的动漫主页，但手动找封面、整理条目成本高。
- 现有平台多数偏评分、资讯、讨论、社区，不够聚焦“收藏展示”。
- 缺少默认适合分享给别人看的动漫收藏主页。

### 2.4 核心价值
- **低成本收藏**：用户通过搜索即可添加动漫，系统自动匹配官方封面和基础信息。
- **高展示感**：通过封面墙和分区陈列，让收藏行为有“展示柜”般的满足感。
- **易传播**：通过公开主页、链接和二维码，用户可以把自己的动漫收藏分享给别人。
- **国际化**：支持多语言切换，降低不同语言用户的使用和访问门槛。

---

## 3. 产品目标
### 3.1 核心目标
帮助用户快速建立一个属于自己的动漫收藏柜主页。

### 3.2 第一阶段成功标准
用户可以在较短时间内完成：
- 注册
- 添加若干部动漫
- 形成结构清晰、可访问、可分享的个人主页

### 3.3 第一阶段最重要的用户价值
不是交流，不是打分，不是写评论，而是：
**快速把“我喜欢和收藏的动漫”整理成一个可展示的空间。**

---

## 4. 目标用户
### 4.1 核心用户
普遍二次元用户。

### 4.2 用户特征
- 日常会看动漫，但不一定是极硬核追番党。
- 想要整理自己的动漫收藏。
- 对个人主页、封面墙、陈列感、展示感有明显兴趣。
- 愿意把自己的收藏页分享给朋友、群聊或社交平台。

### 4.3 典型用户画像
#### 画像 A：普通动漫爱好者
- 会持续看动漫，也会补经典老番。
- 希望能标记自己“收藏过什么、喜欢什么”。
- 重视简单易用。

#### 画像 B：展示型收藏用户
- 喜欢封面墙、排名、主页展示。
- 更容易被“看起来很酷”的产品打动。
- 更愿意长期维护自己的主页内容。

---

## 5. 用户核心需求
1. 能快速搜索并添加一部动漫到自己的收藏中。
2. 添加时尽量自动匹配动漫的官方封面和基础信息。
3. 能按状态整理动漫，如：已看完、在追、想看。
4. 能手动整理自己最喜欢的番剧排名。
5. 能拥有一个默认公开、可直接分享的个人主页。
6. 能让访客一眼感受到“这个人收藏了很多动漫”。
7. 网站支持多语言切换，方便不同语言环境使用和访问。

---

## 6. 产品边界
### 6.1 本阶段要做
- 动漫搜索与添加
- 自动匹配官方封面和基础信息
- 收藏状态管理
- 喜欢番剧排名
- 公开个人主页
- 主页链接与二维码分享
- 多语言切换
- 基础个人资料管理

### 6.2 本阶段明确不做
- 逐集进度记录
- 当前看到第几集
- 自动判断是否看完
- 用户评分
- 用户短评
- 评论区
- 论坛/讨论组
- 私信
- 关注关系
- 社区互动能力
- 更新时间提醒
- 新番时间表
- 漫画 / Galgame / 轻小说等非动漫内容

### 6.3 为什么不做这些
因为第一阶段需要聚焦在最有差异化、最容易形成产品认知的部分：
**收藏 + 展示 + 分享**。
过早加入社区、评论、进度记录，会让产品变重，也会弱化“收藏柜”这个核心心智。

---

## 7. 核心使用场景
### 场景 1：快速建立收藏
用户搜索一部动漫，系统自动带出封面和基础信息，用户一键加入自己的收藏列表。

### 场景 2：整理收藏结构
用户将动漫归类到“已看完 / 在追 / 想看”等分区中，形成有结构的收藏柜。

### 场景 3：表达个人偏好
用户手动设置自己最喜欢的番剧排名，用于展示个人口味。

### 场景 4：分享给别人看
用户生成个人主页二维码或链接，分享给朋友、群聊或社交平台。

### 场景 5：访客浏览主页
访客进入主页后，可以快速看到该用户收藏了哪些动漫、偏好哪些作品，以及收藏的整体风格。

### 场景 6：多语言访问
来自不同语言环境的用户可以切换界面语言浏览和使用网站。

## 9. MVP 功能需求
### 9.1 账号系统
- 注册
- 登录
- 基础账号管理

### 9.2 个人资料
用户可以编辑：
- 头像
- 名字
- 生日
- 个性签名

### 9.3 多语言切换
支持以下界面语言：
- 简体中文
- 繁體中文
- English
- 日本語
- 한국어

### 9.4 动漫搜索与添加
- 搜索动漫条目
- 查看基础信息
- 添加到个人收藏
- 自动匹配官方封面
- 自动补充基础信息（名称、年份、类型等）
- 设置收藏状态：已看完 / 在追 / 想看 / 搁置 / 弃番

### 9.5 收藏列表管理
- 已看完列表
- 在追列表
- 想看列表
- 按添加时间展示
- 展示封面、名称、添加时间
- 支持状态调整

### 9.6 喜欢番剧排名
- 用户手动选择并排序喜欢的番剧
- 在个人主页显著展示
- 作为个性表达的重要模块

### 9.7 个人主页（核心）
- 默认公开
- 可通过链接访问
- 可生成二维码
- 访客无需登录即可访问
- 重点展示：
  - 头像、名字、生日、个性签名
  - 已看完
  - 在追
  - 想看
  - 喜欢番剧排名
- 整体页面强调“动漫收藏柜”观感
- 使用大量封面墙增强视觉冲击

### 9.8 分享能力
- 复制主页链接
- 生成主页二维码
- 支持将主页作为个人二次元名片进行展示

## 10. 页面级要求
### 10.1 首页
目标：让用户快速理解产品价值并愿意注册。

建议内容：
- 一句话介绍
- 核心卖点
- 示例收藏柜展示
- 注册 / 登录入口
- 多语言入口

### 10.2 搜索页
目标：让用户低成本建立自己的收藏。

展示内容：
- 搜索框
- 搜索结果列表
- 封面、名称、年份、类型
- 快捷添加按钮
- 自动匹配封面的结果呈现

### 10.3 动漫详情页
目标：提供该动漫的基础展示与收藏入口。

展示内容：
- 封面
- 名称
- 简介
- 年份
- 季度（如有）
- 类型
- 收藏状态设置入口

### 10.4 我的收藏页
目标：帮助用户维护自己的收藏结构。

展示内容：
- 已看完 / 在追 / 想看分区
- 封面墙展示
- 添加时间
- 状态调整
- 快速进入详情页

### 10.5 喜欢番剧排名管理页
目标：帮助用户建立“最喜欢的番剧”展示。

展示内容：
- 已收藏动漫选择器
- 拖拽排序或上下移动
- 排名预览

### 10.6 个人主页（动漫收藏柜）
目标：成为用户最核心的展示资产。

展示重点：
- 大量封面墙
- 明确分区感
- 喜欢番剧排名模块
- 基础个人资料
- 分享入口与二维码

视觉目标：
- 像收藏柜、陈列墙、展示册
- 有视觉冲击，但不过于杂乱
- 一眼能看出“这是动漫收藏主页”

## 11. 关键体验原则
1. **添加门槛低**：尽可能少操作地完成一部动漫的收藏。
2. **自动补全信息**：尽量减少用户手动找图、填资料的成本。
3. **展示优先**：主页必须比普通列表更有陈列感。
4. **视觉吸引强**：封面墙是关键体验资产。
5. **结构清晰**：已看完、在追、想看要区分明确。
6. **天然可传播**：公开主页和二维码分享是核心能力。
7. **非社区化**：不靠评论讨论驱动产品。
8. **多语言友好**：适合不同语言用户访问。
9. **PC 优先**：先保证电脑端展示完整和好看。

## 12. 关键用户流程
### 12.1 新用户首次使用
1. 注册账号
2. 完善头像、名字等资料
3. 搜索并添加第一部动漫
4. 系统自动匹配封面和基础信息
5. 选择收藏状态
6. 连续添加多部动漫
7. 自动形成收藏柜主页
8. 复制链接或二维码进行分享

### 12.2 日常使用
1. 登录
2. 搜索新动漫加入收藏
3. 调整状态分类
4. 更新喜欢番剧排名
5. 浏览自己的主页效果

### 12.3 分享传播
1. 进入个人主页
2. 点击分享入口
3. 复制链接或生成二维码
4. 将主页分享到外部渠道
5. 访客打开后浏览该用户收藏

### 12.4 多语言切换
1. 进入网站
2. 点击语言切换
3. 选择对应语言
4. 网站界面切换为目标语言

## 16. 当前阶段建议的增强方向（不含年度/季度报告）
### 16.1 更适合当前产品方向的增强功能
#### 方向 1：收藏墙成就感强化
做法示例：
- 自动生成“收藏墙”视觉页
- 展示收藏总数
- 展示最近新增收藏
- 展示喜欢榜单 Top 区域

价值：
用户来到这个产品，核心诉求是“看见自己的收藏”。强化收藏墙的视觉满足感，会直接提升留存和分享意愿。

#### 方向 2：主页模板与主题皮肤
做法示例：
- 展柜型
- 海报墙型
- 收藏册型
- 极简型

价值：
让主页更像“自己的空间”，提高用户维护主页和持续使用的意愿。

#### 方向 3：收藏维度统计
做法示例：
- 收藏总数
- 类型占比
- 年代分布
- 喜欢榜单构成

价值：
帮助用户理解自己的口味，提升主页深度与自我表达感。

#### 方向 4：轻任务与轻成就
做法示例：
- 添加第一部动漫
- 完成 10 部收藏
- 完善头像、签名、主页资料
- 建立喜欢榜 Top 10

价值：
帮助新用户完成冷启动，提高首周建立收藏柜的概率。

### 16.2 暂不建议优先做的功能
- 年度 / 季度收藏报告
- 评论区
- 社区讨论
- 私信
- 复杂关注关系
- 泛娱乐内容扩展
- 逐集追番记录

原因：
这些功能会让产品过早变重，削弱“收藏 + 展示 + 分享”的主心智。

---

## 17. 当前结论
这个产品第一阶段最应该牢牢守住的不是“功能多”，而是三个字：
**好收藏、好展示、好分享。**

只要把这三件事做得比别的平台更顺、更美观、更有个人空间感，这个产品就能建立清晰差异化。
